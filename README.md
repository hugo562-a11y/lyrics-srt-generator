# 歌詞 SRT 產生器

Windows 桌面工具：匯入音檔，以本機 AI 模型轉成帶時間的歌詞，補上前奏／間奏／尾奏標記，並輸出 SRT。

## 安裝

需要 Python 3.10 以上。首次開啟程式會檢查 `faster-whisper`、`soundfile`、`numpy`、`Pillow`，缺少時會自動下載安裝；本機模型會在第一次分析時下載，之後可離線使用。

```powershell
py -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python app.py
```

Windows 使用者可直接雙擊 `run.bat`，它會建立專案專用的 `.venv` 環境並啟動程式。

## GPU（選用）

預設是「自動（GPU 優先）」。程式會在分析前確認 CUDA 12 的 `cublas64_12.dll` 與 `cudnn64_9.dll` 都能被目前 Python 行程找到；缺少時會自動從 PyPI 安裝對應 NVIDIA 執行庫並加入 DLL 搜尋路徑。其他 GPU 問題則自動回退 CPU，不會中斷分析。

若要發佈免安裝 Python 的 Windows 版本，在已完成首次啟動後執行 `build_exe.bat`；輸出於 `dist\LyricsSrtGenerator\`。請一併發佈整個資料夾，而不是只發佈 EXE。

若音檔格式無法讀取，請安裝並加入 PATH：[FFmpeg](https://ffmpeg.org/download.html)。

## 使用方式

1. 按「匯入音檔」。
2. 選擇模型與語言（中文歌曲建議 `large-v3` 或 `medium`；較快可選 `small`）。
3. 按「開始 AI 分析」。模型在本機執行，會列出每句歌詞與音樂區段。
4. 可直接雙擊表格中的開始、結束、類型或文字修改；選取一列可用工具列新增或刪除。選取一句歌詞、把播放頭移到想斷開的位置，按「✂ 在此斷句」可把一句拆成兩句（時間與文字都依比例自動分配）。`Ctrl+Z` / `Ctrl+Y` 可復原／重做。
5. 按「匯出 SRT」；若要直接用於影片剪輯，按「匯出動態 PNG（透明）」；若要做卡拉OK版本，按「匯出人聲／伴奏（卡拉OK）」。

完成分析後，使用下方的「播放／暫停／停止」與時間滑桿試聽。播放游標進入任何歌詞、前奏、間奏或尾奏區段時，表格會自動捲動並以藍色標示該列；可在送進 Premiere 前先修正錯位。播放器使用 FFmpeg 隨附的 `ffplay`，不需要額外編譯 Python 音訊套件。

點選表格任一列會從該列開始連續播放；按「只播選取句」則在該列結束時自動停止。手動停止會保留目前播放位置。

表格上方是聲波與時間軸：每句歌詞／音樂段會以色塊顯示在聲波上，開始邊界是綠色、結束邊界是紅色，若 AI 辨識的起訖時間不準可直接拖曳邊界校正（放開滑鼠即套用，可用 `Ctrl+Z` 復原）；點色塊或表格列只會選取該句並移動播放頭，**不會自動開始播放**，方便單純點選、拖曳校正而不被打斷（若原本就在播放中，點選會直接接續從新位置播放）；點上方時間尺可直接跳轉播放位置。滾輪可以游標所在時間點為中心縮放，按住滑鼠中鍵拖曳可平移時間軸；右上角也可用 ＋／－縮放或按「符合視窗」。此功能需要 `ffmpeg.exe` 在 PATH 內，若找不到只會顯示提示，不影響其他功能。

按空白鍵可隨時播放／暫停（焦點在文字輸入框時空白鍵仍正常輸入文字，不會誤觸播放）；要單獨試聽某一句可按「▶ 只播選取句」。

### 動態字幕 PNG 序列

「匯出動態 PNG（透明）」會使用**目前表格中未刪除的歌詞列**及其已校正的時間。「字幕樣式與匯出」面板可選輸出比例：16:9（1920×1080）、9:16（1080×1920）、1:1（1080×1080）、4:3（1440×1080）；全部為 30 fps RGBA PNG 影像序列。可選四種動畫：逐字點亮、彈跳聚焦、滑入淡出、電影柔和。字級大小、文字顏色、外框顏色、垂直／水平位置（上/中/下、左/中/右）與左右／上下偏移都可自訂，右側「字幕預覽」面板會即時顯示效果，且**跟隨主播放器同步**（播放音樂時字幕動畫會一起跑，拖曳時間軸或點表格列也會立刻更新預覽畫面），不需要另外開視窗、不會輸出檔案或改動時間軸。前奏、間奏、尾奏不會顯示文字，但仍保留透明影格，因此可從第一張直接對齊音檔時間軸。直式及方形畫面會自動依安全區調整中文字級與換行。輸出完成後，在 Premiere、DaVinci Resolve 等軟體選擇「影像序列」並指定 30 fps 匯入即可。

### 卡拉OK人聲／伴奏分軌

「匯出人聲／伴奏（卡拉OK）」用本機 Demucs 把目前音檔分成人聲、伴奏兩個獨立 WAV 檔（`<檔名>_人聲.wav`、`<檔名>_伴奏.wav`），存到你指定的資料夾，方便另外拿去做卡拉OK版本或混音。首次使用會下載 Demucs 模型，需要一些時間；之後會使用本機快取。

介面採用暗色主題，比照主流影音剪輯工具的風格。

若已有正確歌詞，請先按「匯入歌詞檔」。使用 UTF-8 `.txt` 或 `.lrc`，**每句歌詞一行**；LRC 原有時間碼會忽略，改以實際音檔重抓節奏。AI 只用於找出實際演唱的時間與節奏，匯出的文字會採用歌詞檔原文，不會採用 AI 聽錯的字。

### 提升對齊準確度

- 保持勾選「精準逐字對齊」：Whisper 會產生逐字時間錨點，再把正確歌詞逐行套上，通常比整句時間範圍精準。
- 匯入正確歌詞時，程式會關閉一般語音的靜音過濾，避免拖音或伴奏中的後段歌詞被過早忽略；對齊也會同時約束每句字數，避免文字在後段突然跑完。
- 程式另外以保守的語音偵測尋找第一句人聲；若前方空檔達到「最短音樂段」設定，會保留為 `[前奏]`，而不會被伴奏誤當成歌詞。
- 伴奏較大聲、合唱或拖音明顯時，勾選「先分離人聲」：程式會按需下載 Demucs 與其本機模型，先從音檔抽出人聲再對齊。第一次較久，之後模型會快取。
- 人聲分離是選用功能；若其相依套件無法在目前 Python 版本安裝，關閉勾選仍可使用逐字對齊。

## 標記規則

- 第一個歌詞前：`[前奏]`
- 歌詞句與句間的音樂空檔：`[間奏]`
- 最後一句後：`[尾奏]`
- 較短的呼吸／停頓不會產生 SRT 項目；可在介面調整「最短音樂段」。

SRT 是 `HH:MM:SS,mmm` 規格，因此第一筆從 `00:00:00,000`（等同所要求的 `00:00:00:00` 起點）開始。
